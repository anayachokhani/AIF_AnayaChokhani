# Repository Structure

The task plan was written for a conventional `frontend/` plus `backend/`
layout. This project uses the OpenAI Sites/vinext scaffold, where the working
Next.js frontend lives at the repository root under `app/`.

Mapping:

- Planned `frontend/app/` -> actual `app/`
- Planned `frontend/components/` -> actual `app/components/`
- Planned `frontend/lib/` -> actual shared frontend data/helpers in `app/data.ts`
- Planned `backend/formaos/` -> actual `backend/formaos/`
- Planned `docs/` -> actual `docs/`
- Planned `data/` -> actual `data/`
- Planned `artifacts/` -> actual `artifacts/`

The `frontend/` folder is retained with README placeholders so a new teammate
can reconcile the task plan with the deployed Sites scaffold.
